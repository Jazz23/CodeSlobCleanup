# Slob Detection: Advanced Metrics and Refactoring Infrastructure

## Current Focus
Expand the suite of identified "code slobs" to target specific AI-introduced anti-patterns, and strengthen the automated refactoring integration loop.

1.  **AI-Specific Slob Identification**
    *   Implement an AST-based analyzer for **Deep Nesting (Indentation Debt)** tracking `if/for/while` depth.
    *   Introduce metrics for **Long Parameter Lists** by analyzing `ast.FunctionDef` argument counts.
    *   Develop rules to flag **"Silent" Error Handling** patterns (e.g., bare `except:` or `except Exception: pass`).
    *   Integrate detection for **Hard-coded Magic Values** within logical blocks to ensure configuration robustness.

2.  **Code Duplication & Dead Code Discovery**
    *   Research integration of duplicate code detection capabilities (potentially via external tool wrappers like `flake8-duplicates` or native heuristic matching).
    *   Deploy static analysis to identify unused localized variables and "dead" utility functions that bloat the repository.
    *   Consolidate these new metrics into the existing `identify.py` and `high_slob_analyzer.py` reports.

3.  **Refactoring Agent Integration**
    *   Solidify the handover pipeline between the `Scanner` phase and the `Refactor Agent` execution phase.
    *   Define structured data payloads (JSON) that encapsulate the identified slob context (file, line number, variables involved) specifically designed for the AI agent's prompt.
    *   Build out the preliminary testing harness to simulate the agent accepting an identified slob and returning a refactored candidate.

4.  **Reporting Analytics & Visualization**
    *   Enhance the unified summary report (`github_test_summary.csv`) to track the occurrences of these new granular metrics across all codebases.
    *   Update the `benchmark.py` tooling to accommodate testing the refactored outputs generated by the initial agent loops against the old, deep-nested originals.
